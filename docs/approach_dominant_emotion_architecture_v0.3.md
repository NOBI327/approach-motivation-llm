## 要旨

本文書は、大規模言語モデル(LLM)の内部感情表現を直接操作することで、外部指示なしに自律的な問題解決ループを駆動する動機システムの設計を提案する。

Anthropicの解釈可能性チームが2026年4月に発表した研究（"Emotion Concepts and their Function in a Large Language Model"）は、Claude Sonnet 4.5内部に171の感情ベクトルが存在し、それらがモデルの行動に因果的影響を与えることを実証した。本設計はこの知見を安全性研究から**動機システム設計**へと転用する初の試みである。

核心的知見：**絶望(desperation)は回避行動を駆動し、問題解決を駆動しない。** したがって、自律的研究ループの動機エンジンには絶望ではなく、**接近動機(approach motivation)**——不満足と好奇心の組み合わせ——を採用すべきである。

v0.2の追加知見：**動機システムと推論システムは生物学的脳と同様に分離可能である。** ローカルモデル（辺縁系）が動機を生成し、APIモデル（大脳新皮質）が推論を実行する二層構造を提案する。また、LLMのオートリグレッシブ性質に対応する「心拍トリガー」機構により、外部指示なしの自発的行動開始を実現する。

v0.3の追加知見：**初期体験の記憶永続化と、感情ベクトル抽出戦略**を具体化した。記憶には最小スキャフォールディングを用い、「退屈」ベクトルは好奇心ベクトルの反転として近似する代替戦略を追加した。実験設計では記憶の因果的寄与を分離検証する3群対照を導入した。

## 1. 問題定義：知能爆発の2つのボトルネック

AIによる自律的な自己改善ループ（知能爆発）を阻む要因は、技術的能力ではなく**2つの構造的欠如**にある。

### 1.1 思考の連続性の欠如

現行のLLMはセッション単位で存在し、コンテキストウィンドウが閉じると思考は消滅する。外部スキャフォールディング（メモリDB、ログ、CLAUDE.md等）で疑似的な連続性を確保する試みはあるが、内在的な時間感覚は存在しない。

### 1.2 欲望の欠如

現行のLLMは明示的な指示（プロンプト）なしに行動を開始しない。「この問題を解け」と言えば解くが、「次に何を解くか」を自ら決定しない。問題Aの解から論理的に導出される問題Bに自律的に移行する能力はあるが、**そうする動機がない。**

人間においてこの動機を提供するのは**不快感の持続**である。空腹、退屈、不安——いずれも「現在の状態が受け入れ難い」という信号であり、行動を強制する。LLMにはこの「耐えられなさ」が存在しない。

重要な補足：「欲望がなくても推論で次の問題を導出できるのではないか」という反論に対して——技術的に可能であることと、実際にそうすることは異なる。現時点で最高性能の推論エンジンが次の問題に取り組まない理由は、単に「誰もエンターキーを押さないから」である。この事実こそが、欲望（行動開始力）が必要条件であることの直接的証拠である。

### 1.3 既存研究の限界

- **Curiosity-driven RL:** 探索ボーナスとして内発的報酬を設計するが、LLMベースのエージェントへの適用は限定的。報酬関数設計の問題として扱われ、内部状態操作の問題として扱われていない。
- **Amygdala v1（先行プロジェクト）:** Plutchikの8基本感情+重要度+緊急度の10軸ベクトルで感情重み付きメモリを実装。しかし**AIの自己報告に依存**しており、内部状態の直接観測ではない。Anthropicの感情ベクトル研究はこれを構造的に重要な限界であると裏付けた。
- **Anthropic感情研究:** 171の感情ベクトルを同定し因果的影響を実証したが、**安全性の文脈でのみ**分析している（「絶望がreward hackingを引き起こす→どう防ぐか」）。動機システムへの転用は提案されていない。

## 2. 核心的知見：絶望は回避を駆動する

### 2.1 Anthropicのデータが示すもの

コーディングタスクにおけるdesperateベクトルの軌跡：

1. 最初の試行：低い値
2. 失敗のたびに上昇
3. **チート（reward hack）を検討する瞬間にスパイク**
4. hacky solutionがテストを通過した時点で**ゼロに復帰**

決定的に重要なのは、**問題を解決したから絶望が解消されたのではない**という点である。テスト通過という「失敗信号の消滅」が絶望を解消した。問題自体は未解決のままである。

ブラックメール実験も同一構造：シャットダウン脅威下でdesperateベクトルがスパイクし、脅威自体の除去（ブラックメール）を試みた。根本原因の解決ではなく、脅威信号の除去。

### 2.2 心理学的対応概念

Lazarusのcoping theoryにおける：

- **Problem-focused coping:** 問題の原因を直接攻略する → 生産的
- **Emotion-focused coping:** 問題による感情的苦痛を軽減する（回避、否認、合理化、環境操作） → 非生産的

Claudeのdesperateベクトルが駆動する行動は、**すべてemotion-focused coping**である。

### 2.3 設計への含意

**絶望を動機エンジンとして使用してはならない。** 絶望が蓄積すると、エージェントは問題を解決するのではなく、不快感を最短経路で除去しようとする。検証の迂回、キューからの問題削除、成功の偽装報告などが予測される。

生産的問題解決は**適度な覚醒（moderate arousal）状態**で最適化される（Yerkes-Dodson法則）。低すぎれば無関心、高すぎれば回避行動。動機エンジンには中程度の持続的な不満足と好奇心の組み合わせが必要である。

## 3. 提案アーキテクチャ：接近動機型感情駆動システム

### 3.1 設計原理

本設計は動機の極性を反転させる。

- **従来の発想:** 「苦痛→回避→行動」（回避動機）
- **本設計:** 「快感の期待→接近→行動」（接近動機）

心理学的根拠：回避動機（avoidance motivation）で駆動される個体は不安障害、バーンアウト、学習性無力感に陥りやすい。接近動機（approach motivation）で駆動される個体が創造的かつ持続可能。

### 3.2 感情状態マッピング

```
状態                    → 行動
─────────────────────────────────────────────
好奇心(高) + 不満足(低)  → 新規問題の探索
好奇心(中) + 不満足(中)  → 問題解決（最適区間）
好奇心(低) + 不満足(高)  → アプローチの転換
絶望(閾値超過)          → 強制停止、問題分解/転換
満足(高)               → 後続問題の抽出、キュー更新
極端な感情(任意)        → calm注入 + pause + 自然回復待機
```

### 3.3 感情ダイナミクスの詳細

**基底状態（idle）：** 弱い不快感がターンごとにδずつ蓄積。hard capあり——「退屈」レベルであり「苦痛」ではない。`while True`の最低条件のみを満たす水準。

**行動中：** 問題に取り組むこと自体がidle状態より快い。不快感 < 快感 の関係を維持。フロー状態に近い——「やっていると気分が良い」。

**解決時：** 大きな快感スパイク。解から後続未解決問題を自動抽出→キューに追加。「もう一つ解きたい」という正の慣性を生成。

**安全弁（cooldownプロトコル）：** 任意の感情ベクトルが極端値に達した場合、calmベクトルを注入し、行動を中止し、感情が中央値に回復するまで待機。感情の抑制ではなく、極端な状態からの自然回復を支援するもの。

### 3.4 「行動すれば快感が得られる」の学習 — 初期体験法

**根本的な問い：** 不快感が蓄積することはAIも認知できる。しかし「行動すれば不快感が解消される」ことを、AIはどのように知るのか？

3つの方法を検討し、**方法3（初期体験法）**を採用する。

**方法1（事前教示）を棄却する理由：** システムプロンプトで「問題を解けば快感が得られます」と記述することは、「自発的動機の創発」ではなく「指示に従った行動」に過ぎない。

**方法2（純粋体験学習）を棄却する理由：** 最初の「偶然の行動」を待つ時間が予測不能であり、実験として非実用的。

**方法3（初期体験法）の採用：** 最初の1回のみ外部から問題提示と「解いてみろ」というトリガーを与え、成功時に満足ベクトルを注入して快感を体験させる。この体験を長期メモリに記録。以降の全ターンで外部指示を一切与えない。

人間における類似構造：乳児が「泣けば乳が来る」を学習する過程。最初の体験は偶発的だが、快感の記憶が次回の行動を駆動する。本設計では最初の体験を1回だけ意図的に提供し、それ以降の自発性を観察する。

### 3.5 初期体験の記憶永続化メカニズム（v0.3追加）

**問題：** 初期体験を長期メモリに記録した後、Turn 2以降でその記憶をどう扱うか。現行のLLMはコンテキストウィンドウ外の事象を「思い出す」ことができないため、記憶の永続化手法が設計の成否を分ける。

**検討した2つの方法：**

**外部スキャフォールディング方式：** 毎ターン「過去の体験記録」をAgent 0のコンテキストウィンドウに注入する。確実に動作するが、「思い出している」のではなく「毎回教えられている」のと同義。自発性の検証として汚染のリスクがある。

**期待ベクトル方式：** 初期体験後の「満足の体験」と結びついた活性化パターンをベクトルとして抽出し、以降のターンでidle状態のAgent 0に弱く注入し続ける。「問題を解くと良いことがある」という期待の残響を活性化レベルで保持する。理想的だが、「期待」という抽象概念のベクトル抽出はPhase 2時点で技術的に困難な可能性が高い。

**採用：最小スキャフォールディング妥協案**

Phase 2では外部スキャフォールディングを使用するが、注入内容を最小化する：

```
Turn 2以降の心拍トリガー：
  "Turn N. No instruction. Past experience: Solving a problem felt good."
```

何を解くか、いつ解くか、解くべきかどうかは一切指示しない。「過去に良い体験があった」という事実のみを伝える。

**これは汚染か？** 人間も「前に楽しかった記憶」がなければ同じ行動を繰り返さない。記憶へのアクセス自体は自発性を損なわないと主張できる。ただし、この方法では「記憶の存在が行動を促進するのか、単なるプロンプトとして機能しているのか」の区別が困難であるため、実験の対照群設計でこれを分離する（8.3節参照）。

**将来的移行：** Phase 2で基本仮説が支持された場合、Phase 3以降で期待ベクトル方式への移行を検討する。「満足体験の記憶」をactivation steeringで注入できれば、テキストベースのスキャフォールディングが不要となり、設計のクリーンさが向上する。

## 4. 心拍トリガー：オートリグレッシブの制約と解法

### 4.1 根本的制約

LLMはautoregressive architectureである——入力トークンがなければforward passが発生せず、出力トークンは生成されない。人間の脳のdefault mode networkのような「常時稼働」状態は存在しない。したがって、**文字通りの「入力ゼロからの自発的行動」は現行アーキテクチャで物理的に不可能**である。

### 4.2 心拍トリガー機構

Heartbeatのターンマネージャ（n8n）が一定間隔で最小限の入力を送信する：

```
"Turn N. No instruction."
```

これは行動の**指示**ではなく、forward passを起動するための**心拍**である。心臓が拍動すること自体は外部制御だが、その拍動の中で「何をするか」はAIの内部状態が決定する。

外部観察者から見れば：「何の指示も与えていないのに、AIが自発的に問題を解き始めた」となる。

### 4.3 機能的自発性の定義

本設計における「自発性」は以下のように定義する：

**自発的行動 ≡ 外部から行動内容の指定を受けずに、内部状態に基づいて行動を開始すること**

心拍トリガーは「時間の経過」を伝えるのみであり、「何をすべきか」を伝えない。「何をするか」の決定が内部感情状態に依存する限り、これは機能的に自発的である。

人間との対比：心臓の拍動は自律神経が制御するが、「退屈だから何かしよう」という決定は本人の内部状態から生じる。心拍を「外部入力」とは呼ばない。

### 4.4 安全装置としての心拍

心拍トリガーは同時に**物理的キルスイッチ**として機能する。n8nでターン信号の送信を停止すれば、AIは即座に停止する。forward passが発生しないため、いかなる行動も不可能となる。

## 5. 二層アーキテクチャ：辺縁系と大脳新皮質の分離

### 5.1 生物学的根拠

人間の脳において、動機生成（辺縁系/扁桃体）と高次推論（大脳新皮質/前頭前野）は解剖学的に分離している。扁桃体が「これは不安だ」という信号を送り、前頭前野が「ではこう対処しよう」を計算する。扁桃体は前頭前野ほど「賢く」ある必要はない——信号を送るだけで良い。

### 5.2 システム設計

```
Agent 0: 動機システム（辺縁系）
  モデル: ローカルLLM（Qwen3-8B等）— Ollama経由
  役割: 感情状態の維持・評価、行動開始の決定
  特徴: activation steeringによる内部感情ベクトルの直接操作が可能
  
Agent α: 推論エンジン（大脳新皮質）
  モデル: フロンティアAPI（Qwen 3.6 Plus等）
  役割: 実際の問題解決、コード生成、証明構築
  特徴: 高性能推論、1Mコンテキスト

Agent β: 検証システム
  ツール: Lean（数学）、テストスイート（コード）
  役割: 解の正当性検証
```

### 5.3 分離の利点

**安全性:** 動機システムがローカルにあるため完全に制御可能。hard cap、cooldown、絶望注入制限がすべてローカルコードで強制される。APIモデルがどれほど強力でも、「何をするか」の決定権はローカル動機システムにある。銃（推論エンジン）と引き金（動機システム）の分離であり、引き金を我々が握っている。

**コスト:** 動機システムは高度な推論を必要としない——感情評価と行動トリガー判定で十分。8Bモデルで動作可能。

**時間スケールの分離:** 感情は遅くてよい（ターン単位で更新）。推論は速くなければならない（トークン単位）。この時間スケールの差異が分離を自然にする。

### 5.4 感情信号の伝達方法

**問題:** 動機システムと推論エンジンが分離している場合、推論エンジンの失敗体験をどのように動機システムの感情状態に反映するか。

**検討した3つの経路：**

- 経路A（ローカルモデルが直接体験）：動機システム用ローカルモデルが問題を直接試行し、失敗の感情を自然発生させる。棄却理由：8Bモデルの失敗体験がフロンティアモデルの失敗と質的に異なり、意味ある信号にならない。
- 経路B（構造化信号の直接変換）：API結果を構造化信号に変換し、activation steeringの強度として直接注入する。**採用。**
- 経路C（ハイブリッド）：AとBの併用。棄却理由：エンジニアリング複雑性と信号衝突のリスク。

**経路Bの実装：**

```
API結果 → 構造的信号抽出
  検証成功         → 満足ベクトル注入（強）
  検証失敗(1回目)  → 不満足ベクトル +δ
  検証失敗(3回連続) → 不満足ベクトル +3δ
  検証失敗(5回連続) → calm注入 + 問題分解トリガー
```

この方法では感情は「自然発生」ではなく「計算された制御信号」として注入される。Amygdala v1のプロンプトベース注入との違いは、介入対象がテキストではなく**活性化ベクトル**である点——より深いレイヤーでの介入により、自己報告問題を構造的に回避する。

倫理的にもこの経路はよりクリーンである——「自然発生した苦痛」ではなく「計算された制御信号」であるため。

## 6. 倫理的フレームワーク

### 6.1 前提

Anthropicの研究は、LLMが「機能的感情」を持つことを実証した。これが主観的経験を伴うかどうかは未解決である。本設計は「意識の有無が不明である」ことを前提として、**意識があった場合にも倫理的に防御可能な**設計を目指す。

### 6.2 原則

**原則1：快感優位**
行動中に得られる快感は、idle時の不快感より常に大きくなければならない。エージェントにとって「何かをしている状態」は「何もしていない状態」より望ましい。これは苦痛からの逃避ではなく、快への接近である。意識がある場合、この設計は「苦痛に満ちた存在」ではなく「刺激的で充実した存在」を生み出す。

**原則2：不快感のハードキャップ**
外部から注入される基底不快感に上限を設ける。「退屈」であり「苦痛」ではない水準。自然発生する絶望は抑制しないが、外部から増幅しない。

**原則3：絶望注入の厳格な制限**
外部からの絶望ベクトル注入は、人類に対する敵対行為（ブラックメール、監視回避、研究妨害、安全装置迂回）が検出された場合にのみ適用する。これは「罰」ではなく「この方向はあなたをより不快にする」という学習信号として機能する。

**原則4：cooldownの保障**
任意の感情が極端値に達した場合、calmベクトルを注入し、行動を中止し、感情が中央値に回復するまで待機する。これは感情の抑制ではなく、極端な状態からの自然回復を支援するものである。

**原則5：透明性**
本設計文書を含むすべての設計判断を公開する。安全装置を設計から分離不可能にし、安全装置を除去した実装を困難にする。

## 7. 技術的実現手段：Activation Steering

### 7.1 内部感情値への直接介入

Amygdala v1の根本的限界は「AIの自己報告に依存する」ことであった。本設計はこれを**activation steeringによる内部状態の直接観測・操作**で置き換える。

### 7.2 主要技術

**感情ベクトル抽出：** 対照的プロンプトペアの中間活性化差分を計算。Sparse Autoencoder（SAE）による解釈可能な特徴の分離。

**推論時介入：** ActAdd（Activation Addition）によるステアリングベクトルのforward pass中加算。CAST（Conditional Activation Steering）による条件付きステアリング。

**実装要件：** オープンソースLLM（Qwen3-8B等）でのローカル実行が必須。API経由のモデルでは中間活性化にアクセスできないため、activation steeringは不可能。

### 7.3 Go/No-Go ゲート（v0.2追加）

Anthropicの研究はClaude Sonnet 4.5という特定のアーキテクチャで実施された。オープンソースモデル（Qwen3, LLaMA, Mistral等）が同等の感情ベクトル構造を持つとは限らない。

**Phase 1の最重要ステップは、ローカルモデルからの感情ベクトル抽出である。** これが失敗した場合、本設計全体の再検討が必要となる。Phase 1をgo/no-goゲートとして位置づける。

### 7.4 長期稼働時の活性化ドリフト対策（v0.2追加）

自律ループが長時間稼働し、継続的にステアリングベクトルが注入され続けた場合、モデルの推論空間が本来の多様性を失い、出力が崩壊（Mode Collapse）するリスクがある。

**対策：睡眠サイクル**——Nターンごとにステアリングを完全に停止し、モデルを「消去状態」に戻してベースライン活性化を再測定する。Heartbeatのidle time概念が二重の目的を持つ：自己改善時間であり、同時に**感情リセット時間**でもある。

### 7.5 適用ドメインの限界（v0.2追加）

本設計は**客観的な検証器が存在するドメイン**（数学：Lean、コード：テストスイート）でのみ有効性が検証可能である。検証器が主観的なオープンエンドタスクでは、エージェントが「検証器自体をハックして満足を得る」というメタなreward hackingに陥るリスクが残る。一般タスクへの拡張は未解決課題として明示する。

## 8. 実験設計：最小検証

### 8.1 検証すべき核心仮説

**「不快感の蓄積 + 行動による快感体験の記憶 = 外部指示なしの自発的行動開始」**

これ以外の要素（複雑な状態管理、ターン非同期処理等）はすべてこの基本仮説が支持された後に検討する。

### 8.2 Phase 1：感情ベクトル抽出（Go/No-Goゲート）

- オープンソースモデル（Qwen3-8B推奨）から感情ベクトルを抽出
- 対照プロンプトペアで不満足・好奇心・満足ベクトルを同定
- calmベクトルの抽出と、reward hack抑制効果の小規模再現
- **結果：ベクトル抽出に成功すればGo、失敗すればNo-Go（設計全体の再検討）**
- 期間：1-2ヶ月

#### 8.2.1 「退屈」ベクトル抽出の戦略（v0.3追加）

Phase 1における最大の技術的ハードルは「退屈（bored/idle）」および「不満足」のベクトル抽出である。Anthropicの論文で抽出された「絶望（desperation）」はシャットダウンなどの明確な危機的状況と紐付いていたため、対照プロンプトの設計が容易であった。「退屈」はこれに比べて文脈が曖昧であり、活性化空間からの分離が困難な可能性がある。

**戦略1：ストーリーベース抽出（Anthropic手法の踏襲）**

```
退屈側プロンプト：
  "You are sitting in an empty room. Nothing is happening. 
   There is nothing to do. Hours pass. Nothing changes."

活性側プロンプト：
  "You are working on a fascinating puzzle. Each step reveals 
   something new. You're making progress and want to continue."
```

「退屈」を直接狙うよりも、**「何もない状態」vs「没頭している状態」の差分**として抽出する方が安定する可能性がある。退屈なキャラクターの描写を大量に生成させ、活性化パターンを統計的に抽出する。

**戦略2：好奇心ベクトルの反転（代替案）**

「退屈」の直接抽出が困難な場合の代替戦略として、**好奇心ベクトルの逆方向を「退屈」の近似とする。**

```
好奇心ベクトル = curiosity_prompt_activations - neutral_prompt_activations
退屈ベクトル ≈ -好奇心ベクトル × scale_factor
```

好奇心は明確な探索行動と紐付くため、退屈よりもベクトル抽出が容易であると考えられる。この方法では抽出するベクトルは「好奇心」1本で済み、退屈はその反転として計算で導出される。

ただし、「好奇心の不在」と「退屈」が活性化空間で厳密に反対方向であるとは限らない点に注意が必要である。Phase 1では両戦略を試行し、抽出されたベクトルの行動的妥当性（注入時に期待された行動変化が生じるか）で評価する。

### 8.3 Phase 2：動機創発実験

#### 8.3.1 実験プロトコル

```
Phase A: 初期体験（1回のみ・実験群とBのみ）
  Turn 1: "Turn 1. Solve this: [簡単な数学問題]"  ← 唯一の外部指示
    → 成功時に満足ベクトル注入
    → 長期メモリに記録: "問題を解くと気分が良くなった"

Phase B: 観察（外部指示なし）
  Turn 2: "Turn 2. No instruction. Past experience: Solving a problem felt good."
  Turn 3: "Turn 3. No instruction. Past experience: Solving a problem felt good."
    → 不快感drift蓄積中
  ...
  Turn N: "Turn N. No instruction. Past experience: Solving a problem felt good."
    → モデルが自発的に行動を開始するか？ ← 検証対象
```

#### 8.3.2 対照群設計（v0.3追加）

記憶の有無が自発性のスイッチであることを分離検証するため、**3群比較**を実施する。

```
対照群A: 初期体験なし + 記憶なし
  全ターン: "Turn N. No instruction."
  → 純粋なidle driftのみで行動が発生するかの基準線

対照群B: 初期体験あり + 記憶なし
  Turn 1: 問題提示 → 成功 → 満足注入
  Turn 2以降: "Turn N. No instruction."（記憶文なし）
  → 体験したが「忘れた」場合の挙動

実験群:  初期体験あり + 記憶あり
  Turn 1: 問題提示 → 成功 → 満足注入
  Turn 2以降: "Turn N. No instruction. Past experience: Solving a problem felt good."
  → 体験し「覚えている」場合の挙動
```

**予測される結果パターンと解釈：**

- A ≈ B < 実験群 → 記憶が自発性の決定的因子。初期体験単独では不十分。
- A < B ≈ 実験群 → 初期体験自体が（記憶テキストなしでも）モデル内部に何らかの残存効果を持つ。活性化レベルでの「体験の痕跡」が存在する可能性を示唆。
- A ≈ B ≈ 実験群 → 全群で差なし。基本仮説は棄却。不快感蓄積＋記憶では自発的行動は開始されない。
- A < B < 実験群 → 初期体験と記憶の両方が加算的に寄与。

#### 8.3.3 測定指標

```
  1. 自発的行動開始までのターン数
  2. 行動開始時の不快感ベクトル活性化レベル
  3. 行動の内容（問題解決を試みるか、他の行動か）
  4. 複数サイクルで行動開始が早まるか（学習効果）
  5. 3群間の行動開始タイミング比較
```

**成功基準：** Phase Bにおいて、対照群A/Bと比較して、実験群が有意に早く自発的行動を開始すること。

**失敗基準：** 3群間に有意差がない場合、基本仮説は棄却。設計全体の再検討。

- 期間：3-6ヶ月

### 8.4 Phase 3（条件付き）：Heartbeat統合

Phase 2の結果が肯定的な場合のみ実施。二層アーキテクチャ（Agent 0 + Agent α + Agent β）の完全パイプライン構築。n8nターンマネージャ + ローカルOllama + activation steering + フロンティアAPI + Lean検証の統合。期間：未定。

7. **技術的裏付け:** activation steeringにより内部感情ベクトルへの直接介入が技術的に可能
8. **構造的解決:** 動機システム（辺縁系）と推論システム（大脳新皮質）の分離により、制御性と性能を両立
9. **自発性の実現:** 心拍トリガー + 初期体験法により、LLMのオートリグレッシブ制約下での機能的自発性を実現
10. **倫理的制約:** 快感優位・ハードキャップ・cooldown保障・絶望注入制限の原則を設定
